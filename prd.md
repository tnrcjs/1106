# 구구단 산성비 게임 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트 이름
**구구단 산성비 게임 (Multiplication Rain)**

### 1.2 프로젝트 설명
초등학교 5학년 학생들이 재미있게 곱셈 구구단을 학습하고 연습할 수 있는 교육용 게임입니다. 하늘에서 떨어지는 곱셈 문제를 제한 시간 내에 정답을 입력하여 막아내는 아케이드 스타일의 게임입니다.

### 1.3 배경 및 필요성
- 구구단은 수학의 기초이지만 반복 학습이 지루할 수 있음
- 게임화(Gamification)를 통해 학습 동기 부여
- 즉각적인 피드백으로 학습 효과 극대화

---

## 2. 목표 및 목적

### 2.1 핵심 목표
- 초등학교 5학년 학생들의 곱셈 구구단 숙달도 향상
- 반복 학습을 통한 자동 암산 능력 배양
- 재미있는 학습 경험 제공

### 2.2 학습 목표
- 2단부터 9단까지 곱셈 구구단 완전 숙달
- 빠른 계산 능력 향상
- 집중력 및 순발력 향상

---

## 3. 타겟 사용자

### 3.1 주요 타겟
- **연령**: 초등학교 5학년 (만 10-11세)
- **수준**: 기본 구구단을 배운 상태이나 숙달이 필요한 학생
- **사용 환경**: 학교 컴퓨터실, 가정 PC, 태블릿

### 3.2 사용자 특성
- 게임에 익숙한 세대
- 짧은 집중 시간 (10-15분)
- 즉각적인 보상과 성취감 선호
- 시각적 피드백에 민감

---

## 4. 핵심 기능

### 4.1 게임 모드

#### 4.1.1 연습 모드
- **목적**: 특정 단 집중 연습
- **특징**:
  - 2단 ~ 9단 중 선택 가능
  - 제한 시간 없음
  - 실수해도 게임 오버 없음
  - 정확도와 속도 기록

#### 4.1.2 도전 모드
- **목적**: 전체 구구단 실전 연습
- **특징**:
  - 랜덤으로 2단 ~ 9단 문제 출제
  - 3개의 생명 (하트)
  - 점수 및 레벨 시스템
  - 속도가 점차 증가

#### 4.1.3 시간 제한 모드
- **목적**: 빠른 계산 능력 테스트
- **특징**:
  - 2분 제한 시간
  - 최대한 많은 문제 해결
  - 콤보 시스템으로 추가 점수

### 4.2 게임 메커니즘

#### 4.2.1 기본 게임 플레이
1. 화면 상단에서 곱셈 문제가 적힌 빗방울이 떨어짐
   - 예: "3 × 7", "8 × 4"
2. 플레이어는 화면 하단 입력창에 정답 입력
3. 정답 입력 시:
   - 해당 빗방울 소멸 (파티클 효과)
   - 점수 획득
   - 효과음 재생
4. 오답 입력 시:
   - 빗방울 계속 낙하
   - 입력창 흔들림 효과
5. 빗방울이 바닥에 닿으면:
   - 생명 1개 감소
   - 바닥에 물웅덩이 생성 (시각적 피드백)

#### 4.2.2 난이도 조절
- **Level 1-3**: 1개의 빗방울, 느린 속도
- **Level 4-6**: 2개의 빗방울, 보통 속도
- **Level 7-9**: 3개의 빗방울, 빠른 속도
- **Level 10+**: 4개의 빗방울, 매우 빠른 속도

#### 4.2.3 점수 시스템
- 기본 점수: 10점
- 콤보 보너스: 연속 정답 시 점수 배율 증가
  - 5콤보: 1.5배
  - 10콤보: 2배
  - 20콤보: 3배
- 빠른 정답 보너스: 빗방울이 화면 상단 30% 영역에 있을 때 정답 시 추가 5점

### 4.3 보상 시스템

#### 4.3.1 스티커 수집
- 각 단별 마스터 스티커 (90% 이상 정확도)
- 레벨 달성 스티커
- 특별 업적 스티커 (예: 50콤보 달성)

#### 4.3.2 캐릭터 및 테마 언락
- 점수를 모아 다양한 캐릭터 잠금 해제
- 배경 테마 변경 (우주, 바다, 숲 등)

## 5. UI/UX 요구사항

### 5.1 화면 구성

#### 5.1.1 메인 화면
```
[로고]
[시작하기]
[모드 선택: 연습 | 도전 | 시간제한]
[내 기록]
[설정]
```

#### 5.1.2 게임 화면
```
┌─────────────────────────────────────┐
│ ❤❤❤  Level 5  Score: 1250  콤보: 7  │
├─────────────────────────────────────┤
│                                     │
│         [7×8]  (빗방울1)            │
│                                     │
│                    [4×6] (빗방울2)  │
│                                     │
│  [캐릭터 이미지]                     │
│                                     │
│  ┌────────────────────────┐        │
│  │   답: [입력창]   [확인]  │        │
│  └────────────────────────┘        │
└─────────────────────────────────────┘
```

### 5.2 디자인 가이드라인

#### 5.2.1 색상
- **주색상**: 밝은 파란색 (#4A90E2) - 하늘/물 테마
- **보조색**: 노란색 (#FFD700) - 정답 효과
- **경고색**: 빨간색 (#FF6B6B) - 오답/위험
- **배경**: 부드러운 그라데이션 (하늘 느낌)

#### 5.2.2 타이포그래피
- **숫자**: 큰 글씨, 읽기 쉬운 고딕체 (30-40pt)
- **UI 텍스트**: 둥근 고딕체 (16-20pt)
- **높은 대비**: 가독성 최우선

#### 5.2.3 애니메이션
- 빗방울 낙하: 부드러운 이동 (ease-in)
- 정답 효과: 반짝임 + 파티클 폭발
- 오답 효과: 화면 흔들림 (shake)
- 레벨업: 축하 애니메이션

### 5.3 사운드

#### 5.3.1 효과음
- 정답: 맑은 종소리 (Ding!)
- 오답: 부드러운 경고음 (Buzz)
- 바닥 충돌: 물방울 소리
- 콤보: 점점 높아지는 톤
- 레벨업: 팡파레

#### 5.3.2 배경음악
- 경쾌하고 반복적인 멜로디
- 레벨에 따라 템포 증가
- 음소거 옵션 필수

## 6. 기술 요구사항

### 6.1 플랫폼
- **주 플랫폼**: 웹 브라우저 (HTML5)
- **호환성**: Chrome, Firefox, Safari, Edge 최신 버전
- **반응형**: 데스크톱 (1024x768 이상), 태블릿 (768x1024)

### 6.2 기술 스택 (권장)

#### 6.2.1 프론트엔드
- **HTML5**: 구조
- **CSS3**: 스타일링 및 애니메이션
- **JavaScript (ES6+)**: 게임 로직
- **Canvas API** 또는 **Phaser.js**: 게임 렌더링

#### 6.2.2 데이터 저장
- **LocalStorage**: 사용자 점수 및 진행도 저장
- **(선택) 백엔드**: 리더보드 구현 시 필요

### 6.3 성능 요구사항
- **프레임률**: 60 FPS 유지
- **로딩 시간**: 3초 이내
- **반응 속도**: 입력 후 50ms 이내 피드백

### 6.4 접근성
- 키보드 입력 지원 (Enter로 답변 제출)
- 숫자 키패드 지원
- 색맹 모드 옵션
